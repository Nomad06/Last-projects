Утилита для сохранения в локальный файл веб-страницы или файла с любых интернет-сайтов путем соединения с ними по высокоуровневым протоколам.

Через командную строку можно задать 3 параметра: (1)полное имя ресурса, (2)директория сохранения файлов(если не указать, то сохраняется в папку с проектом),
(3)автооткрытие локальной копии страницы после создания(задаётся через 0 или 1)

Если URL содержит лишь имя домена (например, http://mail.ru), то имя файла фиксировано: например, index.html;
Если URL содержит параметры (например, http://go.mail.ru/realtime?q=java), то символ ? и следующие за ним параметры игнорируются в имени файла
Если сохраняемый ресурс является html-страницей, то в поддиректорию с именем вида имя_файла_files сохраняются имеющиеся на странице картинки (указанные в тегах img), 
а также другие (указанные в тегах link) ресурсы, необходимые для корректного отображения страницы. При этом содержимое html-страницы при сохранении меняется – вместо 
полного пути к таким ресурсам в тегах img и link указывается локальный путь, начинающийся с "имя_файла_files/".
Автоматически определяет кодировку ресурса и использует эту кодировку при чтении страницы для корректной загрузки русских и т.п. символов.
